Working with frameworks


In the previous Tutorial, we learned how a AsyncTask works. Here in this section, we will learn how to implement AsyncTask by building a simple app as shown in the video above.

Creating Project :

Open Android Studio. Set Application name as AsyncTask. Company domain as com.example.ssame.demo. Click next keeping the project location in the same location. Now choose the phone and tablet by targeting API 22 and later, click next and select the EmptyActivity with Activity name MainActivity.java and layout name activity_main.xml and click finish.

activity_main.xml

In activity_main.xml layout file, change rootview Linearlayout and delete the layout constraints setAttribute if present. Add a TetView , Button and a ProgressBar. Add onClick attribute in the button widget as sartTask Now the Yellow highlighted part appeared on the onClick attribute as the startProgress() method is not yet implemented in the MainActivity.java press ALT+ENTER(Option + Enter on n Mac) placing the cursor on the highlighted text and choose to create 'startTask(View) in the MainActivity.java' to create the method in MainActivity.java.

MainActivity.java

Now in our MainActivity.java file add a variable to store TextView and progressbar. In the oncreate() method, initialize both TextView and Progressbar and make the progressbar invisible.

Async class

create an inner class Async in mainActivity.java extend Async with AsyncTask. AsyncTask takes three generic type parameters:
params:we declared void as this AsyncTask does not require any inputs.
progress :we declared Void as the progress is not published.
result: we Declared String as we will update the TextView with a string when the execution is completed by AsyncTask.


doInBackground()


Now implement doInbackground() method. Press Alt+Enter(option + Enter on Mac) on the red highlighted part where we extended AsyncTask and implement methods click ok. The doInbackground() method is invoked in a background thread. Hence, we can do any long operation. Here we implement the code to execute which makes the thread to sleep for certain time.
In the doInBackground() method Generate a random integer between 1 to 20.Multiply that by 150. The total is the amount of time we put our current thread to sleep using Thread.sleep()

onPostExecute()

when doInbackground() completes the return value is passed to onPostExecute() callback. onPostExecute() method publishes results on the UI. override onPostExecute() method by pressing ctrl+O. Now Here onPostExecute() takes a string argument and display that string in textView.

MainActivity.java

Now add two string fields for Google API which we got from API developer Console and youtube video ID. youtube video ID is nothing but the part after the v in the URL.




In our, MainActivity.java extend YouTubebaseActivity and implement YoutubePlayer.OninitializedListener now Press Alt+Enter(option + Enter on Mac) on the red highlighted part where we extended YouTubeBaseActivity.

The two Methods which will be implemented are:

1.onInitializationSuccess method is called when initialization was sucess.

This method has three parameters :
 1.provider:

The provider which was used to initialize the YouTubePlayer.

2.player :

A YouTubePlayer which can be used to control video playback in the provider.

3:wasRestored:

Whether the player was restored from a previously saved state, as part of the YouTubePlayerView restoring its state.

2.onInitializationFailure is called when the initialization of the player fails.
This method has two parameters :
 1.provider:

The provider which failed to initialize a YouTubePlayer.

2.error :

A YouTubePlayer which can be used to control video playback in the provider.



In the MainActivity oncreate initialize() youtubePlayerview. Now in onInitializationSuccess methods we use two interfaces.

1. YouTubePlayer.PlaybackEventListener

This interface is invoked when there is a video playback event.

2.YouTubePlayer.PlayerStateChangeListener

This interface is invoked when there is a change in their player state.
